This is our API that is created with Python and it handles POST and GET requests with our database

Below i will give a short explaination of the different Routes one can use.

  • /contacts
    This will send a GET request to our database and it will display all the data in the database on the webpage as .json
  • /contacts/vcard
    This will send a GET request to our database and it will display all the data in the database on the webpage as .vcf format
  • /contacts/vcard/json
    This will send a GET request to our database and it will display all the data in the database on the webpage as .vcf format inside a .json structure
  • /contacts/vcard/download
    This will send a GET request to our database, and you can download all the data in our database as a .vcf file
  • /contacts/id
    This will send a GET request to our database and it will display the data associated with that id on the webpage as .json
  • /contacts/id/vcard
    This will send a GET request to our database and it will display the data associated with that id on the webpage as .vcf format
  • /contacts/id/vcard/json
    This will send a GET request to our database and it will display the data associated with that id on the webpage as .vcf format inside a .json structure
  • /upload.html
    This will send a GET request returning the upload.html file, where there is a form that the user is suppose to upload a .vcf file. When the user uploaded a .vcf file and when the user has submitted that file, the API send a POST request to the database adding the data in the .vcf file to the database.
